# Multi-Agent: Progressive Disclosure with Skills

The skills pattern loads domain knowledge on demand rather than including everything in the initial prompt. This keeps the base context small and focused, loading detailed instructions only when the agent encounters a specific domain.

---

## Concept

Instead of stuffing the system prompt with instructions for every possible task, define **skills** -- self-contained knowledge modules that the agent can load when needed. This is progressive disclosure: start broad, get specific only when required.

Benefits:
- Smaller base prompts (fewer tokens, lower cost, less confusion)
- Scales to many domains without ballooning context
- Skills can be updated independently
- Agent only loads what it needs for the current task

---

## Skill Structure

Each skill has a name, description (used for discovery), and content (the detailed instructions):

```python
from dataclasses import dataclass

@dataclass
class Skill:
    name: str
    description: str
    content: str
```

## Defining Skills

### Sales Analytics Skill

```python
sales_analytics_skill = Skill(
    name="sales_analytics",
    description="Analyze sales data, revenue trends, top products, and regional performance",
    content="""
# Sales Analytics Guide

## Available Metrics
- **Revenue**: Total sales amount, broken down by period, region, or product
- **Units Sold**: Count of items sold
- **Average Order Value (AOV)**: Revenue / Number of Orders
- **Conversion Rate**: Orders / Visitors

## Common Analyses
1. **Trend Analysis**: Compare metrics over time (daily, weekly, monthly, quarterly)
2. **Product Performance**: Rank products by revenue, units, or margin
3. **Regional Breakdown**: Compare sales across geographic regions
4. **Customer Segmentation**: Group customers by purchase behavior

## SQL Patterns
- Revenue by month: SELECT DATE_TRUNC('month', order_date) as month, SUM(amount) FROM orders GROUP BY 1
- Top products: SELECT product_name, SUM(quantity) FROM order_items GROUP BY 1 ORDER BY 2 DESC LIMIT 10
- Regional comparison: SELECT region, SUM(amount) FROM orders GROUP BY region

## Interpretation Guidelines
- Always compare to previous period (MoM, QoQ, YoY)
- Flag anomalies (>20% deviation from trend)
- Consider seasonality before drawing conclusions
"""
)
```

### Inventory Management Skill

```python
inventory_management_skill = Skill(
    name="inventory_management",
    description="Manage stock levels, reorder points, warehouse operations, and supply chain",
    content="""
# Inventory Management Guide

## Key Concepts
- **Stock Level**: Current quantity of an item in the warehouse
- **Reorder Point**: Minimum stock level that triggers a new order
- **Lead Time**: Time between placing an order and receiving it
- **Safety Stock**: Buffer inventory to prevent stockouts
- **Economic Order Quantity (EOQ)**: Optimal order quantity to minimize costs

## Formulas
- Safety Stock = Z-score * StdDev(demand) * sqrt(lead_time)
- Reorder Point = (Average Daily Usage * Lead Time) + Safety Stock
- EOQ = sqrt((2 * Annual Demand * Order Cost) / Holding Cost)

## Alert Thresholds
- CRITICAL: Stock < 10% of reorder point
- WARNING: Stock < reorder point
- HEALTHY: Stock > reorder point

## Common Operations
1. Check stock levels for a product or category
2. Calculate reorder recommendations
3. Identify slow-moving inventory (no sales in 90+ days)
4. Track incoming shipments and expected delivery dates
"""
)
```

## Skill Registry and Loading Tool

```python
# Registry of all available skills
SKILL_REGISTRY = {
    skill.name: skill
    for skill in [sales_analytics_skill, inventory_management_skill]
}

from langchain.tools import tool

@tool
def load_skill(skill_name: str) -> str:
    """Load detailed instructions for a specific domain skill.

    Available skills:
    - sales_analytics: Analyze sales data, revenue trends, top products
    - inventory_management: Manage stock levels, reorder points, supply chain
    """
    skill = SKILL_REGISTRY.get(skill_name)
    if not skill:
        available = ", ".join(SKILL_REGISTRY.keys())
        return f"Unknown skill '{skill_name}'. Available skills: {available}"
    return skill.content
```

## Skill Middleware with wrap_model_call

Use middleware to inject loaded skills into the system prompt dynamically:

```python
from langgraph.prebuilt.chat_agent_executor import AgentState
from langgraph.prebuilt import create_react_agent

class SkillState(AgentState):
    """State that tracks which skills have been loaded."""
    loaded_skills: list[str] = []

@wrap_model_call
def skill_middleware(state: SkillState, config):
    """Inject loaded skill content into the prompt."""
    base_prompt = (
        "You are a business intelligence assistant. When a user asks about a specific domain, "
        "first load the relevant skill using the load_skill tool to get detailed instructions.\n\n"
        "Available skills:\n"
    )

    for name, skill in SKILL_REGISTRY.items():
        base_prompt += f"- {name}: {skill.description}\n"

    # Append any already-loaded skill content
    loaded = state.get("loaded_skills", [])
    if loaded:
        base_prompt += "\n--- Loaded Skills ---\n"
        for skill_name in loaded:
            skill = SKILL_REGISTRY.get(skill_name)
            if skill:
                base_prompt += f"\n{skill.content}\n"

    return {"prompt": base_prompt}
```

## Custom State Tracking

Track which skills have been loaded so you can include them in subsequent turns:

```python
from langgraph.types import Command

@tool
def load_skill_with_tracking(skill_name: str) -> Command:
    """Load a skill and track it in state."""
    skill = SKILL_REGISTRY.get(skill_name)
    if not skill:
        available = ", ".join(SKILL_REGISTRY.keys())
        return Command(
            update={},
            # Return error message as tool output
        )

    return Command(
        update={"loaded_skills": [skill_name]},  # Append to loaded skills
    )
```

Alternatively, use a simpler approach where the tool just returns the content and the model incorporates it:

```python
agent = create_react_agent(
    model=llm,
    tools=[load_skill, query_database, generate_chart],
    prompt=(
        "You are a business intelligence assistant. "
        "Before answering domain-specific questions, use load_skill to get "
        "detailed instructions for the relevant domain. "
        "Then use the domain knowledge to provide accurate analysis."
    ),
    state_schema=SkillState,
)
```

## Storage Backends

Skills can be stored in different backends depending on your needs:

- **In-memory dict**: Simple, good for development and small skill sets
- **File system**: One markdown file per skill, easy to edit and version control
- **Database**: SQL or NoSQL, good for dynamic skill management
- **Vector store**: Enable semantic search to find relevant skills automatically
- **LangGraph Store**: Use the namespace/key pattern for persistent cross-session skills

```python
# File-based skill loading
import os

def load_skill_from_file(skill_name: str) -> str:
    """Load a skill from a markdown file."""
    skill_path = os.path.join("skills", f"{skill_name}.md")
    if not os.path.exists(skill_path):
        return f"Skill '{skill_name}' not found."
    with open(skill_path) as f:
        return f.read()
```

## Discovery Methods

How the agent finds the right skill:

1. **Explicit listing**: List all skill names and descriptions in the tool docstring (as shown above)
2. **Semantic search**: Embed skill descriptions and search by query similarity
3. **Hierarchical**: Group skills into categories, search category first, then skill
4. **LLM classification**: Use structured output to classify the query domain

```python
# Semantic discovery with vector store
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()
skill_store = InMemoryVectorStore(embeddings)

# Index skill descriptions
from langchain_core.documents import Document
skill_docs = [
    Document(page_content=skill.description, metadata={"name": skill.name})
    for skill in SKILL_REGISTRY.values()
]
skill_store.add_documents(skill_docs)

# Find relevant skills
def discover_skills(query: str, k: int = 2) -> list[str]:
    """Find the most relevant skills for a query."""
    results = skill_store.similarity_search(query, k=k)
    return [doc.metadata["name"] for doc in results]
```

## Disclosure Strategies

- **Lazy loading**: Load skills only when the agent decides it needs them (most common)
- **Eager loading**: Pre-load skills based on query classification before the agent runs
- **Hybrid**: Pre-load the most likely skill, let the agent load more if needed

---

## Size Considerations

- Keep individual skills under 2000 tokens for best results
- If a skill is too large, split it into sub-skills (e.g., `sales_analytics_queries`, `sales_analytics_interpretation`)
- Use summarized versions for discovery and full versions for execution

## Combining with Few-Shot Prompting

Enhance skills with example interactions:

```python
sales_skill_with_examples = Skill(
    name="sales_analytics",
    description="Analyze sales data and revenue trends",
    content="""
# Sales Analytics Guide
[... core content ...]

## Example Interactions

User: "What was our revenue last month?"
Assistant: Let me query the sales data for last month.
[Runs: SELECT SUM(amount) FROM orders WHERE order_date >= DATE_TRUNC('month', CURRENT_DATE - INTERVAL '1 month')]
Result: $1,234,567
Response: Last month's total revenue was $1,234,567. This represents a 12% increase compared to the previous month.

User: "Which region is underperforming?"
Assistant: I'll compare regional performance against targets.
[Runs: SELECT region, SUM(amount) as actual, target FROM orders JOIN targets USING (region) GROUP BY region, target]
Response: The Northeast region is 23% below target, primarily due to decreased orders in the Electronics category.
"""
)
```
