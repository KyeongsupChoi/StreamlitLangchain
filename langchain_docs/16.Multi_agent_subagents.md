# Multi-Agent: Supervisor with Sub-Agents

The supervisor pattern uses a top-level agent (supervisor) that delegates tasks to specialized sub-agents. Each sub-agent has its own tools, instructions, and domain expertise. The supervisor orchestrates the workflow by choosing which sub-agent to invoke.

---

## Architecture

The architecture has three layers:

1. **API Tools**: Low-level functions that interact with external services (calendar API, email API, etc.)
2. **Sub-Agents**: Domain-specific agents that each own a set of related tools
3. **Supervisor**: Top-level agent that routes user requests to the appropriate sub-agent

```
User <-> Supervisor (Personal Assistant)
              |
         +----+----+
         |         |
    Calendar    Email
     Agent      Agent
      |           |
   [tools]     [tools]
```

## Defining API Tools

```python
from langchain.tools import tool
from datetime import datetime

# Calendar tools
@tool
def list_events(date: str) -> str:
    """List calendar events for a given date (YYYY-MM-DD format)."""
    # In production, call a real calendar API
    return f"Events for {date}: 10am Team Standup, 2pm Design Review, 4pm 1:1 with Manager"

@tool
def create_event(title: str, date: str, time: str, duration_minutes: int = 60) -> str:
    """Create a new calendar event."""
    return f"Created event '{title}' on {date} at {time} ({duration_minutes} min)"

@tool
def delete_event(event_id: str) -> str:
    """Delete a calendar event by ID."""
    return f"Deleted event {event_id}"

# Email tools
@tool
def search_emails(query: str, limit: int = 5) -> str:
    """Search emails by keyword."""
    return f"Found {limit} emails matching '{query}': [1] Project update from Alice, [2] Meeting notes from Bob"

@tool
def send_email(to: str, subject: str, body: str) -> str:
    """Send an email."""
    return f"Email sent to {to} with subject '{subject}'"

@tool
def read_email(email_id: str) -> str:
    """Read the full content of an email."""
    return f"Email {email_id}: From alice@example.com, Subject: Project Update, Body: The project is on track..."
```

## Creating Sub-Agents

Each sub-agent is a self-contained agent with its own tools and instructions.

```python
from langgraph.prebuilt import create_react_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

# Calendar agent
calendar_agent = create_react_agent(
    model=llm,
    tools=[list_events, create_event, delete_event],
    prompt=(
        "You are a calendar management assistant. You help users view, create, "
        "and manage their calendar events. Always confirm actions before making changes. "
        "Use ISO date format (YYYY-MM-DD) and 24-hour time format."
    ),
)

# Email agent
email_agent = create_react_agent(
    model=llm,
    tools=[search_emails, send_email, read_email],
    prompt=(
        "You are an email assistant. You help users search, read, and send emails. "
        "Always confirm the recipient and content before sending an email. "
        "Be concise when summarizing emails."
    ),
)
```

## Wrapping Sub-Agents as Tools

To let the supervisor invoke sub-agents, wrap each one as a tool:

```python
@tool
def calendar_assistant(request: str) -> str:
    """Delegate calendar-related tasks: viewing events, scheduling, rescheduling, or canceling meetings."""
    result = calendar_agent.invoke(
        {"messages": [{"role": "user", "content": request}]}
    )
    return result["messages"][-1].content

@tool
def email_assistant(request: str) -> str:
    """Delegate email-related tasks: searching, reading, composing, or sending emails."""
    result = email_agent.invoke(
        {"messages": [{"role": "user", "content": request}]}
    )
    return result["messages"][-1].content
```

## Building the Supervisor

```python
supervisor = create_react_agent(
    model=llm,
    tools=[calendar_assistant, email_assistant],
    prompt=(
        "You are a personal assistant that helps users manage their calendar and email. "
        "Analyze the user's request and delegate to the appropriate assistant:\n"
        "- Use calendar_assistant for anything related to events, meetings, or scheduling\n"
        "- Use email_assistant for anything related to emails\n"
        "- For requests that span both domains, call each assistant separately\n\n"
        "Synthesize the results into a cohesive response for the user."
    ),
)
```

### Running the Supervisor

```python
# Single-domain request
result = supervisor.invoke(
    {"messages": [{"role": "user", "content": "What meetings do I have tomorrow?"}]}
)
print(result["messages"][-1].content)

# Multi-domain request
result = supervisor.invoke(
    {"messages": [{
        "role": "user",
        "content": "Check my calendar for Friday and email Alice a summary of my schedule"
    }]}
)
print(result["messages"][-1].content)
```

The supervisor will:
1. Call `calendar_assistant` to get Friday's schedule
2. Call `email_assistant` to compose and send the summary email
3. Combine the results into a single response

## Human-in-the-Loop on Sub-Agents

Add approval gates to sensitive sub-agent actions using `HumanInTheLoopMiddleware`:

```python
from langgraph.prebuilt.interrupt import HumanInTheLoopMiddleware
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()

# Add human approval for email sending
email_hitl = HumanInTheLoopMiddleware(
    tools=["send_email"],
    prompt="Review this email before sending:"
)

email_agent_with_approval = create_react_agent(
    model=llm,
    tools=[search_emails, send_email, read_email],
    prompt="You are an email assistant. Always confirm before sending.",
    checkpointer=checkpointer,
)
email_agent_with_approval = email_hitl(email_agent_with_approval)

# Add human approval for calendar modifications
calendar_hitl = HumanInTheLoopMiddleware(
    tools=["create_event", "delete_event"],
    prompt="Review this calendar change:"
)

calendar_agent_with_approval = create_react_agent(
    model=llm,
    tools=[list_events, create_event, delete_event],
    prompt="You are a calendar assistant.",
    checkpointer=checkpointer,
)
calendar_agent_with_approval = calendar_hitl(calendar_agent_with_approval)
```

## Advanced: Controlling Information Flow

### Conversational Context

By default, sub-agents receive only the delegated request string, not the full conversation history. To pass conversation context:

```python
@tool
def calendar_assistant_with_context(request: str, runtime: ToolRuntime) -> str:
    """Delegate calendar tasks with full conversation context."""
    # Include recent conversation history for context
    messages = runtime.state["messages"]
    recent_context = messages[-5:]  # Last 5 messages for context

    context_summary = "\n".join(
        f"{m.type}: {m.content[:200]}" for m in recent_context if m.content
    )

    full_request = f"Context from ongoing conversation:\n{context_summary}\n\nCurrent request: {request}"

    result = calendar_agent.invoke(
        {"messages": [{"role": "user", "content": full_request}]}
    )
    return result["messages"][-1].content
```

### Structured Returns

For tighter integration, sub-agents can return structured data instead of free text:

```python
from pydantic import BaseModel

class CalendarResult(BaseModel):
    success: bool
    action: str
    details: str
    events: list[dict] = []

@tool
def calendar_assistant_structured(request: str) -> str:
    """Delegate calendar tasks and return structured results."""
    result = calendar_agent.invoke(
        {"messages": [{"role": "user", "content": request}]}
    )
    response_text = result["messages"][-1].content

    # Parse or structure the result
    # In practice, you might use structured output from the sub-agent
    return response_text
```

### Limiting Sub-Agent Scope

Prevent sub-agents from going off-track by constraining their tools and instructions:

```python
# Read-only calendar agent for viewing (no create/delete)
readonly_calendar_agent = create_react_agent(
    model=llm,
    tools=[list_events],  # Only the read tool
    prompt="You can only view calendar events. You cannot create or modify them.",
)
```
