# LangGraph Graph API

The Graph API is LangGraph's primary interface for building stateful, multi-step agent workflows. It models computation as a directed graph where **nodes** perform work, **edges** define transitions, and **state** carries data between nodes.

---

## Core Concepts

| Concept   | Description                                                 |
|-----------|-------------------------------------------------------------|
| **State** | A shared data structure passed between nodes                |
| **Node**  | A function that reads state, performs work, and returns updates |
| **Edge**  | A connection defining which node runs next                  |

---

## State Definition

State is the data schema for your graph. It can be defined with `TypedDict`, `dataclass`, or `Pydantic BaseModel`.

### TypedDict

```python
from typing import TypedDict, Annotated
from langgraph.graph import add_messages
from langchain_core.messages import AnyMessage

class AgentState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
    user_name: str
    step_count: int
```

### Dataclass

```python
from dataclasses import dataclass, field

@dataclass
class AgentState:
    messages: Annotated[list[AnyMessage], add_messages] = field(default_factory=list)
    user_name: str = ""
    step_count: int = 0
```

### Pydantic BaseModel

```python
from pydantic import BaseModel, Field

class AgentState(BaseModel):
    messages: Annotated[list[AnyMessage], add_messages] = Field(default_factory=list)
    user_name: str = ""
    step_count: int = 0
```

## Multiple Schema Patterns

Use separate schemas to control what data is visible at different stages:

### Input Schema

Defines what the graph accepts as input. Fields not in the input schema are initialized to defaults.

```python
class InputState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]

class OverallState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
    internal_reasoning: str
    step_count: int

builder = StateGraph(OverallState, input=InputState)
```

### Output Schema

Defines what the graph returns. Internal fields are excluded from the output.

```python
class OutputState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
    final_answer: str

builder = StateGraph(OverallState, input=InputState, output=OutputState)
```

### Private State

Fields in the overall state but not in input or output schemas are private -- used internally by nodes but hidden from callers.

```python
class OverallState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]  # Public
    final_answer: str                                      # Output
    internal_scratchpad: str                                # Private
    retry_count: int                                        # Private
```

## Reducers

Reducers define how state updates are combined when a node returns a value for a field.

### Default Behavior (Override)

Without a reducer, returning a value for a field **replaces** the existing value:

```python
class State(TypedDict):
    value: int  # No reducer: new value overwrites old value

# Node returns {"value": 5} -> state["value"] becomes 5
```

### Custom Reducers with Annotated

Use `Annotated` to attach a reducer function:

```python
from typing import Annotated
import operator

class State(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]  # Append + deduplicate by ID
    scores: Annotated[list[float], operator.add]          # Concatenate lists
    count: Annotated[int, operator.add]                    # Sum integers
```

### The add_messages Reducer

`add_messages` is the most commonly used reducer. It appends new messages and handles deduplication by message ID (updating existing messages if the ID matches).

```python
from langgraph.graph import add_messages

class State(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]

# Appending: node returns {"messages": [new_message]} -> message is appended
# Updating: node returns {"messages": [msg_with_existing_id]} -> existing message is replaced
```

### MessagesState

A prebuilt state class with just a `messages` field using `add_messages`:

```python
from langgraph.graph import MessagesState

class MyState(MessagesState):
    # Inherits: messages: Annotated[list[AnyMessage], add_messages]
    extra_field: str = ""
```

## Nodes

Nodes are functions that receive the current state and return updates.

### Basic Node Structure

```python
def my_node(state: AgentState) -> dict:
    """Process state and return updates."""
    messages = state["messages"]
    last_message = messages[-1]
    # ... do work ...
    return {"messages": [response], "step_count": state["step_count"] + 1}
```

### Injectable Parameters

Nodes can declare additional parameters that are automatically injected:

```python
from langchain_core.runnables import RunnableConfig
from langgraph.store.base import BaseStore

def my_node(state: AgentState, config: RunnableConfig, store: BaseStore) -> dict:
    """Node with injected config and store."""
    user_id = config["configurable"]["user_id"]
    memories = store.search(("memories", user_id), query="relevant info")
    # ...
    return {"messages": [response]}
```

Available injectable parameters:
- `config: RunnableConfig` -- runtime configuration
- `store: BaseStore` -- long-term memory store
- `writer: StreamWriter` -- for streaming intermediate results

### Special Nodes

**START**: The entry point of the graph. Not a real node -- it just defines where execution begins.

**END**: The terminal node. Reaching END stops execution and returns the final state.

```python
from langgraph.graph import START, END

builder.add_edge(START, "first_node")
builder.add_edge("last_node", END)
```

### Node Caching

Cache expensive node results to avoid recomputation:

```python
builder.add_node("expensive_lookup", expensive_function, cache_policy="cache")
```

## Edge Types

### Normal Edges

Unconditional transitions from one node to another:

```python
builder.add_edge("node_a", "node_b")  # Always go from A to B
```

### Conditional Edges

Route to different nodes based on state:

```python
def route_decision(state: AgentState) -> str:
    """Decide which node to go to next."""
    if state.get("needs_review"):
        return "review_node"
    elif state["step_count"] > 5:
        return "summarize_node"
    else:
        return "continue_node"

builder.add_conditional_edges(
    "decision_node",
    route_decision,
    # Optional explicit mapping:
    # {"review": "review_node", "summarize": "summarize_node", "continue": "continue_node"}
)
```

### Parallel Edges (Fan-Out)

Multiple edges from one node cause parallel execution:

```python
builder.add_edge("start_node", "agent_a")
builder.add_edge("start_node", "agent_b")
builder.add_edge("start_node", "agent_c")
# agent_a, agent_b, agent_c all run in parallel
```

Results are collected using the reducers defined in the state.

## Send API for Dynamic Routing

`Send` enables dynamic fan-out where the number of parallel branches is determined at runtime:

```python
from langgraph.constants import Send

def fan_out(state: AgentState) -> list[Send]:
    """Dynamically create parallel branches."""
    topics = state["topics_to_research"]
    return [
        Send("research_node", {"topic": topic, "messages": state["messages"]})
        for topic in topics
    ]

builder.add_conditional_edges("planner", fan_out)
```

Each `Send` creates a separate invocation of the target node with its own input state. Results are merged back using reducers.

## Command Object

`Command` combines state updates with routing instructions in a single return value. Nodes and tools can return `Command` to both update state and control flow.

```python
from langgraph.types import Command

def my_node(state: AgentState) -> Command:
    """Update state and route to a specific node."""
    return Command(
        update={"step_count": state["step_count"] + 1},
        goto="next_node"
    )
```

```python
# Route to multiple nodes in parallel
def split_node(state: AgentState) -> Command:
    return Command(
        update={"status": "splitting"},
        goto=["branch_a", "branch_b"]
    )
```

```python
# Command with Send for dynamic routing
def dynamic_node(state: AgentState) -> Command:
    return Command(
        goto=[
            Send("worker", {"task": task})
            for task in state["pending_tasks"]
        ]
    )
```

## Graph Compilation and Runtime

### Compilation

```python
graph = builder.compile()
```

### With Checkpointer (Persistence)

```python
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)
```

### With Store (Long-Term Memory)

```python
from langgraph.store.memory import InMemoryStore

store = InMemoryStore()
graph = builder.compile(checkpointer=checkpointer, store=store)
```

### Runtime Configuration

```python
config = {
    "configurable": {
        "thread_id": "session-123",
        "user_id": "user-456",
    }
}

result = graph.invoke({"messages": [{"role": "user", "content": "Hello"}]}, config=config)
```

## Context Schema

Define a schema for the context passed at invocation time:

```python
from dataclasses import dataclass

@dataclass
class AppContext:
    user_id: str
    tenant_id: str
    is_admin: bool = False

builder = StateGraph(AgentState, context_schema=AppContext)
```

## Recursion Limits

Prevent infinite loops by setting a recursion limit:

```python
graph = builder.compile()

result = graph.invoke(
    {"messages": [{"role": "user", "content": "Hello"}]},
    config={"recursion_limit": 25}  # Default is 25
)
```

### RemainingSteps

Nodes can check how many steps remain before hitting the limit:

```python
from langgraph.types import RemainingSteps

def my_node(state: AgentState, remaining_steps: RemainingSteps) -> dict:
    """Adapt behavior based on remaining steps."""
    if remaining_steps < 3:
        # Running low on steps, wrap up
        return {"messages": [{"role": "assistant", "content": "Let me summarize what I've found so far."}]}
    # Continue normal processing
    return {"messages": [response]}
```

## Graph Migrations and Compatibility

When you modify a graph's state schema, existing checkpoints may become incompatible. LangGraph supports migrations to handle this:

- **Adding fields**: New fields with defaults are automatically handled
- **Removing fields**: Old checkpoint data for removed fields is ignored
- **Renaming fields**: Requires a migration function

```python
# Adding a new field with a default is backward-compatible
class AgentStateV2(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
    user_name: str
    step_count: int
    new_field: str  # New field, needs a default

# Existing checkpoints without new_field will use the default value
```

For complex migrations, use the checkpoint migration utilities provided by your checkpointer backend.

## Complete Example

```python
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.checkpoint.memory import MemorySaver
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

class State(MessagesState):
    step: str = "initial"

def classify(state: State) -> dict:
    response = llm.invoke([
        {"role": "system", "content": "Classify the user message as 'question' or 'statement'."},
        state["messages"][-1]
    ])
    return {"step": response.content.strip().lower()}

def handle_question(state: State) -> dict:
    response = llm.invoke([
        {"role": "system", "content": "Answer the user's question helpfully."},
        *state["messages"]
    ])
    return {"messages": [response]}

def handle_statement(state: State) -> dict:
    response = llm.invoke([
        {"role": "system", "content": "Acknowledge the user's statement and ask a follow-up."},
        *state["messages"]
    ])
    return {"messages": [response]}

def route(state: State) -> str:
    if "question" in state["step"]:
        return "handle_question"
    return "handle_statement"

builder = StateGraph(State)
builder.add_node("classify", classify)
builder.add_node("handle_question", handle_question)
builder.add_node("handle_statement", handle_statement)

builder.add_edge(START, "classify")
builder.add_conditional_edges("classify", route)
builder.add_edge("handle_question", END)
builder.add_edge("handle_statement", END)

graph = builder.compile(checkpointer=MemorySaver())

result = graph.invoke(
    {"messages": [{"role": "user", "content": "What is LangGraph?"}]},
    config={"configurable": {"thread_id": "demo"}}
)
print(result["messages"][-1].content)
```
