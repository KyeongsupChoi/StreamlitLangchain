# RAG (Retrieval-Augmented Generation)

RAG combines a retrieval system with a language model so the agent can answer questions grounded in specific documents. The architecture has two phases: **indexing** (prepare and store documents) and **retrieval/generation** (find relevant documents, then generate an answer).

## Indexing Pipeline

The indexing pipeline transforms raw documents into searchable vector embeddings: **Load -> Split -> Store**.

### 1. Load Documents

```python
from langchain_community.document_loaders import WebBaseLoader
import bs4

loader = WebBaseLoader(
    web_paths=("https://lilianweng.github.io/posts/2023-06-23-agent/",),
    bs_kwargs=dict(
        parse_only=bs4.SoupStrainer(
            class_=("post-content", "post-title", "post-header")
        )
    ),
)
docs = loader.load()
```

### 2. Split Documents

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200,
    add_start_index=True,
)
all_splits = text_splitter.split_documents(docs)
print(f"Split into {len(all_splits)} chunks")
```

### 3. Store in Vector Store

```python
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-large")
vector_store = InMemoryVectorStore(embeddings)
_ = vector_store.add_documents(all_splits)
```

## RAG Agent (Tool-Based Retrieval)

The recommended approach wraps retrieval as a **tool** so the agent decides when and how to search. This gives the agent flexibility to reformulate queries, search multiple times, or skip retrieval entirely.

### Define the Retrieval Tool

```python
from langchain.tools import tool

@tool(response_format="content_and_artifact")
def retrieve_context(query: str):
    """Retrieve information related to a query."""
    retrieved_docs = vector_store.similarity_search(query, k=2)
    serialized = "\n\n".join(
        f"Source: {doc.metadata}\nContent: {doc.page_content}"
        for doc in retrieved_docs
    )
    return serialized, retrieved_docs
```

The `response_format="content_and_artifact"` setting returns both a text summary (sent to the model) and the raw document objects (available programmatically).

### Build the Agent

```python
from langgraph.prebuilt import create_react_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

system_prompt = (
    "You are a research assistant. Use the retrieve_context tool to find "
    "relevant information before answering questions. Always cite your sources. "
    "If the retrieved information doesn't contain the answer, say so."
)

agent = create_react_agent(
    model=llm,
    tools=[retrieve_context],
    prompt=system_prompt,
)
```

### Run the Agent

```python
result = agent.invoke(
    {"messages": [{"role": "user", "content": "What is Task Decomposition?"}]}
)

for message in result["messages"]:
    print(f"{message.type}: {message.content[:200]}")
```

## RAG Chain (Without Agent)

For simpler use cases where you always want to retrieve then generate, use a two-step chain.

### Two-Step Chain

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

retriever = vector_store.as_retriever(search_kwargs={"k": 3})

rag_prompt = ChatPromptTemplate.from_messages([
    ("system", "Answer the question based only on the following context:\n\n{context}"),
    ("human", "{question}")
])

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": lambda x: x}
    | rag_prompt
    | llm
    | StrOutputParser()
)

answer = rag_chain.invoke("What is Task Decomposition?")
print(answer)
```

### Single Inference (Generate with Retrieved Context)

For a more explicit approach:

```python
def rag_generate(question: str) -> str:
    """Retrieve context and generate an answer in one function."""
    # Step 1: Retrieve
    docs = vector_store.similarity_search(question, k=3)
    context = "\n\n".join(doc.page_content for doc in docs)

    # Step 2: Generate
    messages = [
        {"role": "system", "content": f"Answer based on this context:\n\n{context}"},
        {"role": "user", "content": question}
    ]
    response = llm.invoke(messages)
    return response.content
```

## Returning Source Documents

To include source metadata alongside the answer, structure your output to carry both:

```python
from pydantic import BaseModel

class RAGResponse(BaseModel):
    answer: str
    sources: list[dict]

def rag_with_sources(question: str) -> RAGResponse:
    """Generate an answer with source attribution."""
    docs = vector_store.similarity_search(question, k=3)
    context = "\n\n".join(doc.page_content for doc in docs)

    messages = [
        {"role": "system", "content": f"Answer based on this context:\n\n{context}"},
        {"role": "user", "content": question}
    ]
    response = llm.invoke(messages)

    sources = [
        {
            "content": doc.page_content[:200],
            "metadata": doc.metadata
        }
        for doc in docs
    ]

    return RAGResponse(answer=response.content, sources=sources)
```

When using the agent-based approach with `response_format="content_and_artifact"`, the raw documents are automatically available in the `ToolMessage.artifact` field:

```python
from langchain_core.messages import ToolMessage

for message in result["messages"]:
    if isinstance(message, ToolMessage) and message.artifact:
        for doc in message.artifact:
            print(f"Source: {doc.metadata.get('source', 'unknown')}")
            print(f"Content: {doc.page_content[:100]}...")
            print()
```
