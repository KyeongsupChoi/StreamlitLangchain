# Semantic Search

Semantic search finds documents by meaning rather than keyword matching. It works by converting text into numerical vectors (embeddings) and finding vectors that are close together in the embedding space.

---

## Documents and Document Loaders

LangChain represents content as `Document` objects with `page_content` (text) and `metadata` (dict).

```python
from langchain_core.documents import Document

doc = Document(
    page_content="LangChain is a framework for building LLM applications.",
    metadata={"source": "docs", "page": 1}
)
```

### Document Loaders

Loaders ingest content from various sources and return lists of `Document` objects.

```python
# PDF loading
from langchain_community.document_loaders import PyPDFLoader

loader = PyPDFLoader("path/to/document.pdf")
pages = loader.load()
# Each page becomes a Document with page number in metadata
```

```python
# Web page loading
from langchain_community.document_loaders import WebBaseLoader

loader = WebBaseLoader("https://example.com/article")
docs = loader.load()
```

Other loaders include:
- **TextLoader**: Plain text files
- **CSVLoader**: CSV files (one document per row)
- **DirectoryLoader**: All files in a directory
- **UnstructuredFileLoader**: PDFs, Word docs, HTML, and more via the Unstructured library
- **GitLoader**: Files from a git repository
- **NotionDBLoader**: Notion database pages

## Text Splitters

Documents often exceed the optimal chunk size for embedding. Text splitters break them into smaller, overlapping pieces.

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200,
    length_function=len,
    separators=["\n\n", "\n", ". ", " ", ""],
    add_start_index=True,
)

chunks = text_splitter.split_documents(pages)
print(f"Split {len(pages)} pages into {len(chunks)} chunks")
```

`RecursiveCharacterTextSplitter` tries to split on the largest separator first (paragraph breaks), then falls back to smaller ones. The `chunk_overlap` ensures context is not lost at boundaries.

Other splitters:
- **CharacterTextSplitter**: Splits on a single separator
- **TokenTextSplitter**: Splits by token count (useful for token-limited models)
- **MarkdownHeaderTextSplitter**: Splits by markdown headers, preserving hierarchy
- **HTMLHeaderTextSplitter**: Splits by HTML headers
- **SemanticChunker**: Splits based on embedding similarity between sentences

## Embeddings

Embedding models convert text into fixed-dimensional vectors. Similar texts produce vectors that are close together.

Supported providers:
- **OpenAI**: `pip install langchain-openai` -- `OpenAIEmbeddings(model="text-embedding-3-large")`
- **Anthropic (Voyage)**: `pip install langchain-voyageai` -- `VoyageAIEmbeddings(model="voyage-3")`
- **Google**: `pip install langchain-google-genai` -- `GoogleGenerativeAIEmbeddings(model="models/embedding-001")`
- **HuggingFace**: `pip install langchain-huggingface` -- `HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")`
- **Cohere**: `pip install langchain-cohere` -- `CohereEmbeddings(model="embed-english-v3.0")`
- **AWS Bedrock**: `pip install langchain-aws` -- `BedrockEmbeddings(model_id="...")`

```python
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-large")

# Embed a single query
query_vector = embeddings.embed_query("What is semantic search?")
print(f"Vector dimension: {len(query_vector)}")

# Embed multiple documents
doc_vectors = embeddings.embed_documents([
    "Semantic search uses embeddings",
    "Keyword search uses exact matching"
])
```

## Vector Stores

Vector stores index embeddings and support efficient similarity search.

Supported stores:
- **InMemoryVectorStore**: Built-in, no external dependencies
- **Chroma**: `pip install langchain-chroma`
- **FAISS**: `pip install langchain-community faiss-cpu`
- **Pinecone**: `pip install langchain-pinecone`
- **PGVector**: `pip install langchain-postgres`
- **Qdrant**: `pip install langchain-qdrant`
- **Weaviate**: `pip install langchain-weaviate`
- **Milvus**: `pip install langchain-milvus`

### Creating and Populating a Vector Store

```python
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-large")
vector_store = InMemoryVectorStore(embeddings)

# Add documents
vector_store.add_documents(chunks)

# Or create from documents in one step
vector_store = InMemoryVectorStore.from_documents(chunks, embeddings)
```

## Querying

### Basic Similarity Search

```python
results = vector_store.similarity_search(
    "What is task decomposition?",
    k=3
)

for doc in results:
    print(f"Content: {doc.page_content[:100]}...")
    print(f"Metadata: {doc.metadata}")
    print()
```

### Similarity Search with Scores

```python
results_with_scores = vector_store.similarity_search_with_score(
    "What is task decomposition?",
    k=3
)

for doc, score in results_with_scores:
    print(f"Score: {score:.4f}")
    print(f"Content: {doc.page_content[:100]}...")
    print()
```

### Search by Vector

```python
query_vector = embeddings.embed_query("What is task decomposition?")
results = vector_store.similarity_search_by_vector(query_vector, k=3)
```

### Async Search

```python
results = await vector_store.asimilarity_search(
    "What is task decomposition?",
    k=3
)
```

## Retrievers

Retrievers provide a standardized interface for fetching documents. They integrate with LangChain's chain and agent abstractions.

### Custom Retriever with @chain

```python
from langchain_core.runnables import chain

@chain
def custom_retriever(query: str) -> list[Document]:
    """Retrieve and re-rank documents."""
    # Get more candidates than needed
    docs_with_scores = vector_store.similarity_search_with_score(query, k=10)

    # Filter by score threshold
    filtered = [
        doc for doc, score in docs_with_scores
        if score < 0.5  # Lower score = more similar for some stores
    ]

    # Return top results
    return filtered[:3]

# Use like any retriever
docs = custom_retriever.invoke("task decomposition")
```

### VectorStore as Retriever

The simplest way to create a retriever from a vector store:

```python
retriever = vector_store.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 4}
)

docs = retriever.invoke("What is task decomposition?")
```

### Search Types

**Similarity** (default): Returns the k most similar documents.

```python
retriever = vector_store.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 4}
)
```

**Maximum Marginal Relevance (MMR)**: Balances relevance with diversity to reduce redundancy in results.

```python
retriever = vector_store.as_retriever(
    search_type="mmr",
    search_kwargs={
        "k": 4,
        "fetch_k": 20,    # Fetch 20 candidates
        "lambda_mult": 0.5  # 0 = max diversity, 1 = max relevance
    }
)
```

**Similarity Score Threshold**: Returns only documents above a relevance threshold.

```python
retriever = vector_store.as_retriever(
    search_type="similarity_score_threshold",
    search_kwargs={
        "score_threshold": 0.8,
        "k": 4
    }
)
```

### Using Retrievers in Chains

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_messages([
    ("system", "Answer based on context:\n\n{context}"),
    ("human", "{question}")
])

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

chain = (
    {"context": retriever | format_docs, "question": lambda x: x}
    | prompt
    | llm
    | StrOutputParser()
)

answer = chain.invoke("What are the approaches to task decomposition?")
```
