# SQL Agent

Build an agent that can query and analyze relational databases using natural language. The agent translates user questions into SQL, executes queries, and returns human-readable answers.

---

## Safety Warning

SQL agents execute model-generated SQL against real databases. This carries inherent risk:

- **Never grant write access** unless absolutely necessary. Use read-only database connections.
- **Scope permissions** narrowly. Restrict the database user to only the tables and operations needed.
- **Add human-in-the-loop** approval for sensitive operations.
- **Consider row limits** to prevent accidentally dumping entire tables.
- **Use query validation** to catch dangerous statements before execution.

---

## Database Setup

This tutorial uses the Chinook sample database (a SQLite database representing a digital media store).

```python
# Download the Chinook database
import urllib.request
url = "https://storage.googleapis.com/benchmarks-artifacts/chinook/Chinook.db"
urllib.request.urlretrieve(url, "Chinook.db")
```

```python
from langchain_community.utilities import SQLDatabase

db = SQLDatabase.from_uri("sqlite:///Chinook.db")

# Inspect the database
print(db.dialect)           # "sqlite"
print(db.get_usable_table_names())
# ['Album', 'Artist', 'Customer', 'Employee', 'Genre', 'Invoice',
#  'InvoiceLine', 'MediaType', 'Playlist', 'PlaylistTrack', 'Track']

# Run a test query
result = db.run("SELECT * FROM Artist LIMIT 5;")
print(result)
```

## SQLDatabaseToolkit

The toolkit provides four tools that give the agent everything it needs to explore and query the database:

```python
from langchain_community.agent_toolkits import SQLDatabaseToolkit
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")
toolkit = SQLDatabaseToolkit(db=db, llm=llm)
tools = toolkit.get_tools()

for tool in tools:
    print(f"{tool.name}: {tool.description[:80]}...")
```

| Tool                    | Purpose                                                  |
|-------------------------|----------------------------------------------------------|
| `sql_db_list_tables`    | List all available tables in the database                |
| `sql_db_schema`         | Get the CREATE TABLE statement and sample rows for specified tables |
| `sql_db_query`          | Execute a SQL query and return results                   |
| `sql_db_query_checker`  | Validate SQL syntax before execution                     |

The agent typically follows this workflow:
1. List tables to understand what is available
2. Get schema for relevant tables
3. Write and validate a SQL query
4. Execute the query
5. Interpret results and answer the user

## Agent Creation

```python
from langgraph.prebuilt import create_react_agent

system_prompt = """You are a database analyst agent. You have access to a SQLite database
with a digital media store's data (artists, albums, tracks, invoices, customers).

Follow these steps for every question:
1. List the available tables using sql_db_list_tables
2. Get the schema for relevant tables using sql_db_schema
3. Write a SQL query to answer the question
4. Validate the query using sql_db_query_checker
5. Execute the query using sql_db_query
6. Interpret the results and provide a clear answer

Important rules:
- Always LIMIT your queries to at most 10 results unless the user specifies otherwise
- Never execute DELETE, UPDATE, INSERT, or DROP statements
- If a query fails, examine the error and try again with a corrected query
- Order results meaningfully (e.g., by count DESC for "most" questions)
"""

agent = create_react_agent(
    model=llm,
    tools=tools,
    prompt=system_prompt,
)
```

### Running the Agent

```python
result = agent.invoke(
    {"messages": [{"role": "user", "content": "Which country's customers spent the most?"}]}
)

for message in result["messages"]:
    if message.content:
        print(f"{message.type}: {message.content[:300]}")
        print()
```

The agent will:
1. List tables and find `Customer` and `Invoice`
2. Get schemas for those tables
3. Write a JOIN query grouping by country with SUM of totals
4. Validate and execute it
5. Report the answer (typically "USA" for the Chinook database)

## Human-in-the-Loop

For production deployments, add human approval before executing SQL queries. LangGraph provides `HumanInTheLoopMiddleware` for this purpose.

```python
from langgraph.prebuilt import create_react_agent
from langgraph.prebuilt.interrupt import HumanInTheLoopMiddleware
from langgraph.checkpoint.memory import MemorySaver

# Define which tools require human approval
hitl = HumanInTheLoopMiddleware(
    tools=["sql_db_query"],  # Only require approval for actual query execution
    prompt="Review this SQL query. Approve or reject:"
)

checkpointer = MemorySaver()

agent = create_react_agent(
    model=llm,
    tools=tools,
    prompt=system_prompt,
    checkpointer=checkpointer,
)

# Apply the middleware
agent = hitl(agent)
```

### Running with Approval

When the agent tries to execute a SQL query, it will pause and wait for human approval:

```python
config = {"configurable": {"thread_id": "sql-session-1"}}

# This will pause when the agent tries to execute SQL
result = agent.invoke(
    {"messages": [{"role": "user", "content": "How many tracks are there?"}]},
    config=config,
)

# The agent is now paused, waiting for approval
# Check what query it wants to run
print("Agent wants to execute:", result)
```

### Resume Execution with Command

After reviewing the proposed query, resume execution:

```python
from langgraph.types import Command

# Approve and continue
result = agent.invoke(
    Command(resume="approve"),
    config=config,
)

print("Final answer:", result["messages"][-1].content)
```

To reject:

```python
result = agent.invoke(
    Command(resume="reject: This query is too broad. Add a LIMIT clause."),
    config=config,
)
```

## Advanced Patterns

### Restricting Available Tables

For large databases, limit the agent to a subset of tables:

```python
db = SQLDatabase.from_uri(
    "sqlite:///Chinook.db",
    include_tables=["Track", "Album", "Artist", "Genre"]
)
```

### Custom Query Validation

Add a validation step beyond the built-in checker:

```python
from langchain.tools import tool

FORBIDDEN_KEYWORDS = ["DELETE", "DROP", "UPDATE", "INSERT", "ALTER", "TRUNCATE"]

@tool
def safe_sql_query(query: str) -> str:
    """Execute a SQL query after safety validation."""
    upper_query = query.upper().strip()

    for keyword in FORBIDDEN_KEYWORDS:
        if keyword in upper_query:
            return f"Error: {keyword} operations are not permitted."

    if "LIMIT" not in upper_query:
        query = query.rstrip(";") + " LIMIT 20;"

    return db.run(query)
```

### Few-Shot Examples

Improve query accuracy by providing example question-SQL pairs in the prompt:

```python
few_shot_examples = """
Example questions and their SQL queries:

Q: How many albums does each artist have?
SQL: SELECT a.Name, COUNT(al.AlbumId) as album_count
     FROM Artist a JOIN Album al ON a.ArtistId = al.ArtistId
     GROUP BY a.Name ORDER BY album_count DESC LIMIT 10;

Q: What are the most popular genres by track count?
SQL: SELECT g.Name, COUNT(t.TrackId) as track_count
     FROM Genre g JOIN Track t ON g.GenreId = t.GenreId
     GROUP BY g.Name ORDER BY track_count DESC;
"""

system_prompt_with_examples = system_prompt + "\n\n" + few_shot_examples
```
