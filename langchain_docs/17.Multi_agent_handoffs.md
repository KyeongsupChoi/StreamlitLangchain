# Multi-Agent: Customer Support with Handoffs

The handoff pattern implements a state machine where the conversation moves between specialized workflow states. Each state has its own prompt, tools, and transition rules. This is ideal for structured processes like customer support, onboarding, or intake workflows.

---

## Architecture

Three workflow states handle a customer support flow:

1. **warranty_collector**: Gathers warranty and product information from the customer
2. **issue_classifier**: Classifies the issue type (hardware, software, user error, etc.)
3. **resolution_specialist**: Provides a resolution based on the classified issue

```
START -> warranty_collector -> issue_classifier -> resolution_specialist -> END
                ^                                          |
                +------------- (backward transition) ------+
```

---

## Custom State

```python
from langgraph.graph import MessagesState
from typing import Optional, Literal

class SupportState(MessagesState):
    current_step: Literal[
        "warranty_collector",
        "issue_classifier",
        "resolution_specialist"
    ] = "warranty_collector"
    warranty_info: Optional[dict] = None
    issue_classification: Optional[str] = None
    resolution: Optional[str] = None
```

## Step Configuration

Define the prompt, tools, and requirements for each step:

```python
STEP_CONFIG = {
    "warranty_collector": {
        "prompt": (
            "You are a warranty information collector. Your job is to gather:\n"
            "1. Product name and model number\n"
            "2. Purchase date\n"
            "3. Warranty ID or proof of purchase\n"
            "4. Brief description of the issue\n\n"
            "Ask for each piece of information if not provided. "
            "Once you have all required info, use the submit_warranty_info tool "
            "to proceed to issue classification."
        ),
        "tools": ["submit_warranty_info"],
        "required_fields": ["product", "purchase_date", "warranty_id", "issue_description"],
    },
    "issue_classifier": {
        "prompt": (
            "You are an issue classifier. Based on the warranty information and "
            "issue description, classify the issue into one of these categories:\n"
            "- hardware_defect: Physical damage or manufacturing defect\n"
            "- software_bug: Software malfunction or crash\n"
            "- user_error: Incorrect usage or configuration\n"
            "- wear_and_tear: Normal degradation over time\n"
            "- incompatibility: Compatibility issue with other products\n\n"
            "Use the classify_issue tool to submit your classification and proceed."
        ),
        "tools": ["classify_issue", "request_more_info"],
    },
    "resolution_specialist": {
        "prompt": (
            "You are a resolution specialist. Based on the issue classification, "
            "provide an appropriate resolution:\n"
            "- hardware_defect: Offer replacement or repair under warranty\n"
            "- software_bug: Provide update instructions or workaround\n"
            "- user_error: Provide step-by-step correct usage instructions\n"
            "- wear_and_tear: Explain warranty limitations, offer discount on replacement\n"
            "- incompatibility: Suggest compatible alternatives\n\n"
            "Use the submit_resolution tool when done. "
            "Use the escalate_back tool if you need more information."
        ),
        "tools": ["submit_resolution", "escalate_back"],
    },
}
```

## State-Transition Tools

Each tool returns a `Command` object that updates the state and routes to the next step.

```python
from langchain.tools import tool
from langgraph.types import Command

@tool
def submit_warranty_info(
    product: str,
    purchase_date: str,
    warranty_id: str,
    issue_description: str
) -> Command:
    """Submit collected warranty information and proceed to issue classification."""
    return Command(
        update={
            "warranty_info": {
                "product": product,
                "purchase_date": purchase_date,
                "warranty_id": warranty_id,
                "issue_description": issue_description,
            },
            "current_step": "issue_classifier",
        },
        goto="agent",  # Route back to the agent node with new step config
    )


@tool
def classify_issue(
    classification: str,
    confidence: float,
    reasoning: str
) -> Command:
    """Classify the issue and proceed to resolution."""
    return Command(
        update={
            "issue_classification": classification,
            "current_step": "resolution_specialist",
        },
        goto="agent",
    )


@tool
def request_more_info(question: str) -> Command:
    """Request more information from the customer. Goes back to warranty collector."""
    return Command(
        update={"current_step": "warranty_collector"},
        goto="agent",
    )


@tool
def submit_resolution(resolution: str, next_steps: str) -> Command:
    """Submit the final resolution to the customer."""
    return Command(
        update={"resolution": resolution},
        goto="__end__",  # End the workflow
    )


@tool
def escalate_back(reason: str) -> Command:
    """Escalate back to issue classification for re-evaluation."""
    return Command(
        update={"current_step": "issue_classifier"},
        goto="agent",
    )
```

## Dynamic Configuration with Middleware

Use `@wrap_model_call` to dynamically configure the model based on the current step:

```python
from langgraph.prebuilt import create_react_agent
from langgraph.prebuilt.chat_agent_executor import AgentState

ALL_TOOLS = [
    submit_warranty_info,
    classify_issue,
    request_more_info,
    submit_resolution,
    escalate_back,
]

def get_tools_for_step(step_name: str) -> list:
    """Return only the tools available in the current step."""
    allowed_tool_names = STEP_CONFIG[step_name]["tools"]
    return [t for t in ALL_TOOLS if t.name in allowed_tool_names]


@wrap_model_call
def dynamic_step_config(state: SupportState, config):
    """Middleware that configures the agent based on the current step."""
    step = state["current_step"]
    step_config = STEP_CONFIG[step]

    # Build context from accumulated state
    context_parts = [step_config["prompt"]]

    if state.get("warranty_info"):
        context_parts.append(f"\nWarranty Info: {state['warranty_info']}")
    if state.get("issue_classification"):
        context_parts.append(f"\nIssue Classification: {state['issue_classification']}")

    system_prompt = "\n".join(context_parts)

    return {
        "prompt": system_prompt,
        "tools": get_tools_for_step(step),
    }
```

## Building the Graph

```python
from langchain_openai import ChatOpenAI
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver

llm = ChatOpenAI(model="gpt-4o")
checkpointer = MemorySaver()

agent = create_react_agent(
    model=llm,
    tools=ALL_TOOLS,
    prompt=dynamic_step_config,
    state_schema=SupportState,
    checkpointer=checkpointer,
)
```

### Running the Support Workflow

```python
config = {"configurable": {"thread_id": "support-ticket-001"}}

# Customer initiates support
result = agent.invoke(
    {"messages": [{"role": "user", "content": "My laptop screen is flickering"}]},
    config=config,
)
print(result["messages"][-1].content)
# Agent asks for warranty details

# Customer provides information
result = agent.invoke(
    {"messages": [{"role": "user", "content": "It's a ThinkPad X1 Carbon, bought Jan 2024, warranty ID WR-12345"}]},
    config=config,
)
print(result["messages"][-1].content)
# Agent submits warranty info and transitions to issue_classifier

# The classifier runs automatically and transitions to resolution_specialist
print(f"Current step: {result.get('current_step')}")
print(f"Classification: {result.get('issue_classification')}")
```

## Message Summarization Middleware

For long support conversations, summarize earlier messages to keep context manageable:

```python
from langchain_core.messages import HumanMessage, AIMessage, SystemMessage

def summarize_messages_middleware(state: SupportState):
    """Summarize conversation history when it gets too long."""
    messages = state["messages"]

    if len(messages) <= 10:
        return messages  # No summarization needed

    # Keep the last 6 messages intact
    recent_messages = messages[-6:]
    older_messages = messages[:-6]

    # Summarize older messages
    summary_text = llm.invoke([
        {"role": "system", "content": "Summarize this conversation concisely:"},
        *[{"role": m.type, "content": m.content} for m in older_messages if m.content]
    ]).content

    summary_message = SystemMessage(content=f"Conversation summary: {summary_text}")
    return [summary_message] + recent_messages
```

## Backward Transitions

The workflow supports backward transitions for cases where later steps need more information. The `request_more_info` and `escalate_back` tools demonstrate this:

```python
# If the issue classifier needs more details:
# classify_issue step -> request_more_info tool -> warranty_collector step

# If the resolution specialist needs re-classification:
# resolution_specialist step -> escalate_back tool -> issue_classifier step
```

This creates a flexible flow where the conversation can move forward and backward through the workflow as needed, while the state accumulates information at each step.

### Transition Diagram

```
warranty_collector --submit_warranty_info--> issue_classifier
issue_classifier --classify_issue---------> resolution_specialist
issue_classifier --request_more_info------> warranty_collector
resolution_specialist --submit_resolution-> END
resolution_specialist --escalate_back-----> issue_classifier
```
